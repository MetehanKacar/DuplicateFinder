# Release Checklist

Use this checklist before creating a public release.

## Product readiness

- [ ] Scan flow tested on sample folders
- [ ] Delete flow tested (Recycle Bin behavior)
- [ ] Swipe mode tested (keyboard + buttons)
- [ ] Language switch tested (TR/EN)
- [ ] No blocking UI regressions

## Repository readiness

- [ ] README is up to date
- [ ] CHANGELOG updated
- [ ] LICENSE present
- [ ] CONTRIBUTING, CODE_OF_CONDUCT, SECURITY present
- [ ] Issue templates and PR template present

## Technical checks

- [ ] Script parses without errors
- [ ] CI workflow passes
- [ ] No accidental debug code left

## GitHub publishing

- [ ] Public repo created
- [ ] Default branch set (`main`)
- [ ] Repo description and topics added
- [ ] Optional: add screenshots/GIFs to README
- [ ] Optional: create first release tag
